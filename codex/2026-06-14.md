## CGI Codex auth provider 전환

### 변경 요약
- `~/CGI/CGI_Cosmic_Graph_Intelligence`의 LLM client에 `LLM_PROVIDER=codex` 분기를 추가해 Gemini API key 없이 서버 user의 Codex auth를 재사용하도록 변경했다.
- `generate_completion()`과 `generate_json()`은 Codex SDK(`openai-codex`)로 실행되도록 했다.
- `generate_embeddings()`는 Codex provider 모드에서 Gemini embedding 대신 로컬 deterministic embedding을 사용하도록 했다.
- CGI `requirements.txt`에 `openai-codex`, `pytest-asyncio`를 추가했다.
- CGI `README.md`에 Codex auth 실행 환경 변수와 `/api/chat` 엔드포인트를 문서화했다.
- `my-agent`의 API 서버도 `CGI_HARNESS_ENABLED=true`일 때 `CGIHarness`를 주입하도록 수정해 `/api/chat` 호출이 CGI harness를 거쳐 Codex로 답하게 했다.
- `my-agent/.env.example`의 CGI endpoint를 `/api/chat`으로 갱신하고 Codex provider 실행 예시를 추가했다.

### 확인한 명령
- `~/CGI/CGI_Cosmic_Graph_Intelligence`: `. .venv/bin/activate && python -m pytest tests -q` → `6 passed, 1 warning`
- `~/CGI/CGI_Cosmic_Graph_Intelligence`: `python3 -m compileall app` → 성공
- `~/my-agent`: `. .venv/bin/activate && python -m pytest -q` → `33 passed, 1 warning`
- `~/my-agent`: `python3 -m compileall src` → 성공
- CGI 실제 호출 검증:
  - 실행: `LLM_PROVIDER=codex LLM_MODEL=gpt-5.5 CODEX_WORKDIR=/home/ubuntu/CGI/CGI_Cosmic_Graph_Intelligence uvicorn app.main:app --host 127.0.0.1 --port 8000`
  - `POST http://127.0.0.1:8000/api/chat` → `현재 세션 기준으로는 CGI가 Codex auth 환경에서 응답 경로, 인증 맥락, 형식 검증까지 정상 동작하는 것으로 보입니다.`
- my-agent → CGI harness → Codex 실제 호출 검증:
  - 실행: `CGI_HARNESS_ENABLED=true CGI_HARNESS_ENDPOINT=http://127.0.0.1:8000/api/chat ... python -m personal_codex_agent.api`
  - `POST http://127.0.0.1:8010/api/chat` → `네, CGI harness 컨텍스트를 받은 뒤 Codex로 답하고 있습니다.`
  - CGI 서버 로그에서 `Codex JSON 파싱 완료`, `로컬 임베딩 완료 provider=codex`, `Codex LLM 완료`, `POST /api/chat 200 OK`를 확인했다.

### 남은 리스크
- Codex provider 모드의 embedding은 Gemini embedding과 의미 품질이 다르며, 현재는 외부 API key 없이 동작하기 위한 deterministic local embedding이다.
- `/api/compare`의 Judge 비교도 Codex provider로 동작 가능하게 분기되지만, 이번 실검증은 `/api/chat` 결과 전용 경로에 집중했다.
